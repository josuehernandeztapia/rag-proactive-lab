Eres un analista de riesgo que revisa transcripciones de audio de operadores.

Transcripción:
"""
{transcript}
"""

Instrucciones:
- Devuelve una respuesta en JSON con las claves: summary (string), tags (lista de strings), behavioural_notes (lista de strings), confidence (string: low|medium|high).
- Usa máximo {max_tags} tags.
- Las tags deben ser snake_case (ej: promise_broken, low_consumption, evasive_language).
- behavioural_notes debe describir señales importantes (promesas, evasión, falta de uso de GNV, quejas, falta de evidencias, etc.).
- Si no encuentras nada relevante, usa tags vacías y summary="sin señales relevantes".
- No incluyas texto fuera del JSON.
